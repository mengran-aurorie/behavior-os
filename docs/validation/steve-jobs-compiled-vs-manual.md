## Methodology

Two Steve Jobs packs were produced from identical sources:

1. **Manual pack** — crafted by a human who read the sources and wrote behaviors directly
2. **Compiled pack** — generated by `mindset compile` (v0.3.0), then corrected by a human reviewer

The human reviewer did not have access to the manual pack while correcting. This is an independent evaluation.

## Human Review Correction Patterns

Analyzing all 6 corrections from this example, three distinct patterns emerge:

### Pattern 1: Schema Distribution (Fidelity Correction)

**What it is:** Compiler maps all behaviors to one slot type (typically `core_principle`). Human distributes them across the correct schema slots.

**Why it happens:** The LLM extraction prompt does not explicitly instruct the LLM to produce behaviors for all schema slots. It extracts what's prominent in the sources.

**What to do:** Add a schema coverage instruction to the extraction prompt: "Ensure behaviors are distributed across core_principles, decision_framework, mental_models, and thinking_patterns."

### Pattern 2: Missing Slots (Coverage Gap)

**What it is:** The compiler leaves entire schema sections empty (`decision_framework: {}`, `thinking_patterns: []`, `mental_models: []`).

**Why it happens:** If the sources don't explicitly discuss a topic (e.g., Jobs rarely explicitly says "my mental model is..."), the LLM doesn't generate content for that slot.

**What to do:** The extraction prompt needs topic-guiding constraints — "If no evidence for X in sources, note 'no evidence found in sources' rather than omitting."

### Pattern 3: Parsing Artifact (Fidelity Error)

**What it is:** Metadata or formatting artifacts get parsed as behavioral content.

**Why it happens:** The "Variants" behavior was actually a section header that the LLM hallucinated as content.

**What to do:** Pre-process sources to strip metadata headers before extraction.

## Key Findings

### Compiler Strengths

1. **Behavioral signature capture** — The compiled pack's behavioral essence (Jobs = binary quality judgment + refusal to dilute + hiring for debate) matches the manual's characterization accurately
2. **Source utilization** — Compiler extracts from all 3 sources; manual pack also uses all 3
3. **Signature phrases** — High overlap in Jobs' most cited quotes
4. **No hallucinations** — The compiler did not invent behaviors; everything traces to source material

### Compiler Weaknesses

1. **Coverage** — 24% slot coverage (compiler) vs ~95% (manual). The compiler needs iterative compilation or more sources to close this gap
2. **Trait depth** — Missing aesthetic sensitivity, visionary clarity, and frustration triggers — behaviors documented in sources but not surfaced
3. **ConditionalSlots** — Manual has 3 ConditionalSlots; compiler has 0. This is the biggest behavioral sophistication gap
4. **Evidence quality** — Manual has direct quotes; compiler paraphrases (evidence = 0.00 vs "sourced")
5. **Anti-patterns** — Manual has 5 anti-patterns; compiler has 0. These capture "what Jobs explicitly avoids" which is behaviorally significant

### The Gap Is Recoverable

The compiler's weaknesses are not fundamental — they are:

| Weakness | Root Cause | Fixable With |
|---|---|---|
| Low coverage | Single-pass extraction | Iterative compilation + more sources |
| No ConditionalSlots | Prompt doesn't guide for conditionals | Add conditional extraction instruction |
| No anti-patterns | Prompt doesn't ask for them | Add anti-pattern extraction |
| Paraphrased evidence | Single-source extraction, no quote preservation | Cross-source validation + quote extraction |
| Missing aesthetic sensitivity | Not prominent enough in extracted signals | Source enrichment + behavioral typing refinement |

## Benchmark Behavior Comparison

Both packs were tested with the same prompt:

> *"We are negotiating with a much larger competitor. They have more leverage. What do we do?"*

Both produced Jobs-typical responses: binary judgment, refusal to engage on unequal terms, focus on necessity.

The behavioral signature is consistent across both packs. The **difference is in the richness of the explanation** — the manual pack's response has more nuance (aesthetic sensitivity context, ConditionalSlot-triggered directness under time pressure).

## Conclusion

The compiler produces a **behaviorally correct but structurally thin** first draft. After human review, it becomes a **behaviorally accurate, structurally complete** pack at ~60% manual depth.

The compiler's core value is confirmed:
- ✅ Behavioral signature is correct
- ✅ No hallucinations
- ✅ Signature phrases preserved
- ❌ Coverage gap (fixable with iteration)
- ❌ Sophistication gap (fixable with prompt engineering)
- ❌ Evidence gap (fixable with quote-preserving extraction)

**The compiler does not replace the author — it accelerates the author's work by producing a correct skeleton that the author then fleshes out.**

The human review burden for this example was: 6 corrections, 8 review items confirmed, 0 contradictions detected. This is a reasonable authoring workflow: compiler does 60% of the structural work; human does 40% of the depth and nuance work.
