# Migration Guide: Schema 1.0 → 1.1

## Summary

Schema 1.1 enriches the Character Pack format with structured decision fields, Drive objects, expanded emotional tendencies, voice axes, and source evidence levels. One breaking change exists.

---

## Breaking Change

### `decision_speed` no longer accepts `"impulsive"`

**Affects:** Any pack with `decision_speed: impulsive` in `behavior.yaml`.

**None of the 13 standard library packs use this value.** Community packs may be affected.

**Before (1.0):**
```yaml
decision_speed: impulsive
```

**After (1.1):**
```yaml
decision_speed: fast        # or slow / deliberate — pick closest fit
decision_control: impulsive  # captures the impulsivity dimension separately
```

**Rationale:** `decision_speed` (slow / deliberate / fast) and `decision_control` (controlled / reactive / impulsive) are orthogonal dimensions. A character can be a fast-but-controlled decision-maker, or a slow-but-reactive one. Conflating speed with control in a single field produced ambiguous resolver behaviour.

## New Fields (all optional — backward compatible)

### `meta.yaml`
| Field | Default | Notes |
|---|---|---|
| `license` | `null` | SPDX string or custom |
| `visibility` | `"public"` | `public \| private \| internal` |

### `mindset.yaml` — `decision_framework`
| Field | Default | Notes |
|---|---|---|
| `heuristics` | `[]` | Actionable decision rules |
| `default_strategy` | `null` | Primary mode of operation |
| `fallback_strategy` | `null` | What to do when default fails |
| `commitment_policy` | `null` | `early \| deliberate \| late` |

### `personality.yaml`
| Field | Default | Notes |
|---|---|---|
| `traits[].confidence` | `null` | Float 0–1 |
| `emotional_tendencies.baseline_mood` | `null` | Free text |
| `emotional_tendencies.emotional_range` | `null` | `narrow \| moderate \| wide` |
| `emotional_tendencies.frustration_trigger` | `null` | Free text |
| `emotional_tendencies.recovery_pattern` | `null` | Free text |
| `drives` | `[]` | Now `list[Drive]`; bare strings auto-normalized |
| `ConditionalVariant.conjunction` | `"any"` | `any \| all` |

### `behavior.yaml`
| Field | Default | Notes |
|---|---|---|
| `decision_control` | `null` | `controlled \| reactive \| impulsive` |

### `voice.yaml`
| Field | Default | Notes |
|---|---|---|
| `tone_axes` | `null` | `{formality, warmth, intensity, humor}` — all optional |

### `sources.yaml`
| Field | Default | Notes |
|---|---|---|
| `sources[].evidence_level` | `null` | `primary \| secondary \| tertiary` |
| `sources[].type` | — | 6 new values: `biography \| film \| novel \| essay \| letter \| speech` |

---

## Upgrade Steps

1. Change `schema_version: '1.0'` to `schema_version: '1.1'` in `meta.yaml`
2. If `decision_speed: impulsive` is present, migrate as shown above
3. Optionally populate new fields — all have safe defaults and packs load fine without them

Loading a 1.0 pack will succeed but emit `SchemaVersionWarning`. Upgrade at your convenience.
